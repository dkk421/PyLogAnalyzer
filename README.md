# PyLogAnalyzer
Backend-сервис для загрузки, парсинга и анализа лог-файлов
## Описание
REST API, позволяющее загружать лог-файлы, асинхронно обрабатывать их и получать аналитическую информацию.
Сервис разделяет этапы загрузки и обработки, что позволяет не блокировать HTTP-запросы при работе с большими файлами.
## Как это работает?

 Клиент загружает лог-файл через API:
1. API сохраняет файл (локально, с возможностью замены на object storage)
2. Создается запись в БД со статусом "uploaded"
3. Запускается фоновая задача обработки
4. Лог парсится построчно
5. Каждая запись сохраняется в БД
6. Статус обновляется: "processing" -> processed / failed"

## Архитектура
Проект построен с разделением ответственности.
 - **router** - HTTP слой (FastAPI)
 - **service** - основная логика
 - **repository** - работа с базой данных

  ### Парсинг логов
  реализован через регулярные выражения.
  Каждая строка преобразуется в структурированный объект.
  Некорректные строки сохраняются как `UNKNOWN`

### Аналитика
- подсчет количества записей по уровням (`ERROR`, `INFO`, `WARNING`)
- получение топа наиболее частых ошибок
- фильтрация записей

## Модель данных
### LogFile:
- id
- filename
- path
- status
- uploaded_at
### LogEntry:
- id
- log_id
- timestamp
- level
- message
- raw_line
  ## Возможности для расширения

Проект изначально спроектирован с учетом масштабирования:
1) переход на S3 / MinIO для хранения файлов
2) использование Celery + Redis для очередей
3) поддержка различных форматов логов
4) добавление метрик и мониторинга
